v-text

  • 设置标签的内容（textContent）
  • 默认写法会替换全部内容，使用差值表达式 {} 可以替换指定内容
  • v-text 指令无论内容是什么，只会解析为==文本==

v-html

  • 设置元素的innerHTML
  • 内容中有html结构会被解析为标签
  • 解析文本使用 v-text，==需要解析html结构==使用 v-html

v-on

  • 为元素绑定事件
  • 指令可以简写为@

绑定事件

v-show

  • 根据布尔值切换元素的显示状态
  • 原理是==修改元素的display==，实现显示隐藏
  • 指令后面的内容最终都会解析为布尔值
  • 数据改变之后，对应元素的显示状态会同步更新

v-if

  • 根据表达式的真假切换元素的显示状态
  • 本质是==通过操纵 dom 元素==来切换显示状态
  • 表达式的值为 true，元素存在于 dom 树中。为 false，从dom树中移除
  • 和 v-show 比较，因为是操作 dom 元素，所以==资源开销较大==
  • 可以做到局部刷新

v-for

  • 根据==数据生成列表结构==
  • 语法是(item,index) in data
  • item 和 index 可以结合其他指令一起使用
  • 数组长度的更新会同步到页面上，是响应式的

作用域插槽

父组件在调用子组件的时候给子组件传了一个插槽，这个插槽为作用域插槽，

该插槽必须放在template标签里面，同时声明从子组件接收的数据放在一个自定义属性内，并定义该数据的渲染方式。

注意点：

  1. ==如果 slot 插槽没有传递参数，slot 会使用默认内容==
  2. 父级模板里的所有内容都是在父级作用域中编译的；子模板里的所有内容都是在子作用域中编译的。
  3. 一个不带 name 的 <slot> 出口会带有隐含的名字“default”。
  4. slot = "btnSlot"等价于#btnSlot
  5. 父组件替换插槽中的标签，子组件提供内容

计算属性

关键字：computed

作用：将复杂的业务逻辑封装并 return，让模板更加简单

注意点：

  1. 计算属性基于 data 中的数据进行处理
  2. 计算属性方法中必须有 return
  3. 使用时不用加括号

计算属性和方法的区别？

  1. 方法不存在缓存
  2. 计算属性是基于它们的响应式依赖进行缓存的

作用：==增加性能，减少开销==

==懒加载==

计算属性只在相关响应式依赖发生改变时它们才会重新求值，

这就意味着只要数据还没有发生改变，多次访问同一个计算属性

计算属性会立即返回之前的计算结果，而不必再次执行函数。

Vuex

Vuex

==state==、==getters==、==mutations==、==actions==、==modules==

每一个 Vuex 应用的核心就是 store（仓库）。“store”基本上就是一个容器，它包含着你的应用中大部分的**状态 (state)**。

Vuex 的状态存储是响应式的。当 Vue 组件从 store 中读取状态的时候，若 store 中的状态发生变化，那么相应的组件也会相应地得到高效更新。

这个状态自管理应用包含以下几个部分：

  • state，驱动应用的数据源；
  • view，以声明方式将 state 映射到视图；
  • actions，响应在 view上的用户输入导致的状态变化。

==管理哪些状态==

登录状态、位置信息、购物车物品
